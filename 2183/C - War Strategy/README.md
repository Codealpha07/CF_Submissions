You, as the country's top general, must strategize where to place your troops.</p><p>There are $$$n$$$ bases in a line, with the $$$k$$$-th of which being the home base for your army. Initially, there is only a single soldier at base $$$k$$$. Each day, the following happens in order:</p><ul> <li> You give out an order by choosing a base $$$i$$$ ($$$1 \leq i \leq n$$$), and <span class="tex-font-style-bf">any</span> number of soldiers inside that base (which is allowed to be $$$0$$$ or all soldiers in that base currently). Then, tell all soldiers you ordered to either move to base $$$i-1$$$ or base $$$i+1$$$. All soldiers must move in the same direction, and no soldier is allowed to move to the left of base $$$1$$$ or to the right of base $$$n$$$. </li><li> Then, a new soldier moves onto base $$$k$$$. This soldier cannot be ordered by that day's commands. </li></ul> <p>However, time is tight, and there are only $$$m$$$ days until the enemy attacks. A base is called <span class="tex-font-style-it">fortified</span> if at least one soldier resides in it. Your job is to find the maximum number of fortified bases (including the home base) you can have by the end of the $$$m$$$-th day.</p></div><div class="input-specification"><div class="section-title">Input</div><p>Each test contains multiple test cases. denoting the number of bases, the number of days you have to fortify your bases, and the index of the home base.</p><p>It is guaranteed that the sum of $$$n$$$ across all test cases does not exceed $$$2\cdot 10^5$$$. </p></div><div class="output-specification"><div class="section-title">Output</div><p>For each test case, print the maximum number of bases you can fortify at the end of the
